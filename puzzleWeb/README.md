# 拼图游戏实现文档

## 一、项目介绍

本拼图游戏是一款基于Web技术栈开发的交互式益智游戏，采用单文件架构设计，整合HTML结构、CSS样式与JavaScript逻辑于puzzle.html文件中。游戏核心目标是通过拖拽与旋转操作，将打乱的图片碎片还原为完整图像，支持2×2至10×10多种难度梯度，适配桌面与移动设备双端操作。

## 二、技术实现

### 2.1 前端架构
页面采用Flexbox弹性布局构建三层结构：顶层为标题与游戏设置容器左右分布，中层为游戏信息记录栏，底层为主游戏区包含拼图框与预览框。CSS运用媒体查询实现响应式适配，确保在768px以下屏幕自动调整布局。配色方案以白色为底色，浅灰(#f8f9fa)为容器背景，蓝色(#007bff)为主按钮色，营造简洁明快的视觉体验。

### 2.2 图片处理机制
游戏使用HTML5 Canvas API进行图像处理。首先计算图片最小边长截取正方形区域，再根据行列参数将图片分割为均等碎片。每个碎片生成独立Canvas元素绘制，通过toDataURL()方法转为Base64格式作为拼图块背景图。针对默认图片test.jpeg实现预加载缓存机制，页面加载时即完成3×3模式的分割缓存，用户点击开始游戏时直接调用缓存数据，显著提升启动速度。

### 2.3 游戏逻辑设计
拼图块采用绝对定位布局，记录correctRow/correctCol标识正确坐标，currentRow/currentCol记录实时位置。游戏状态由isGameStarted布尔值控制，页面初始化时拼图块按正确顺序排列且锁定操作，点击开始游戏后触发scatterPieces()函数随机分布碎片并启动计时器。拖拽功能通过mousedown/mousemove/mouseup事件链实现，计算鼠标偏移量实时更新元素坐标，释放时检测目标位置是否已被占据，触发碎片交换或位置吸附。

### 2.4 旋转与判定系统
每个拼图块初始化时随机赋予0°/90°/180°/270°旋转角度，存储于dataset.rotation属性。双击事件触发rotatePiece()函数，通过CSS transform属性实现旋转动画，同时交换宽高值保持视觉正方形。胜利判定遍历所有拼图块，校验currentRow等于correctRow、currentCol等于correctCol且rotation为0°，三项条件同时满足即判定完成，弹出胜利弹窗显示步数与耗时。

## 三、核心特点

**智能交互设计**：支持鼠标拖拽与触摸屏双模式操作，拼图块靠近正确位置时自动吸附，降低操作精度要求。旋转角度提示默认可选显示，兼顾新手引导与硬核玩家挑战需求。

**性能优化策略**：默认图片预分割缓存避免重复计算，Canvas绘制采用离屏渲染减少重绘开销，transform动画启用GPU硬件加速确保流畅度。

**状态管理严谨**：游戏流程拆分为初始化、进行中、已完成三个阶段，各阶段操作权限严格隔离。信息记录区实时同步移动次数、已放置数量、难度等级与耗时四项数据。

**扩展性架构**：行列数输入框支持2-10范围动态调整，图片上传接口兼容JPEG/PNG格式，代码模块化设计便于后续添加音效、排行榜等功能模块。
